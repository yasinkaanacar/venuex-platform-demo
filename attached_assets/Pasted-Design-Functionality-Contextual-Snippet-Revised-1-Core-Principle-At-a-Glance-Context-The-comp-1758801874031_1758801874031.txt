Design & Functionality: Contextual Snippet (Revised)
1. Core Principle: At-a-Glance Context
The component's goal remains the same: to arm the user with the most critical background information about the review's subject at the exact moment they are about to reply. Removing the manager's name streamlines this focus purely onto operational metrics.

2. Positioning and Layout
The positioning is unchanged. The snippet card appears directly above the full review text within the right-hand reply panel. Its styling remains distinct to separate it from the review content.

3. Dynamic Functionality: Context-Aware Content
The component is still "smart," automatically adapting its content based on the review source.

Scenario A: When a LOCATION Review is Selected (Revised)
The snippet now displays a more focused set of key performance indicators for that specific store, helping the user understand if the current review is an isolated incident or part of a larger trend.

Title: Location Snapshot: [Store Name] (e.g., "Location Snapshot: Ankara Kızılay")

Content Layout: A clean 3-item list for maximum scannability.

Data Points:

Overall Rating: The location's average star rating (e.g., 3.1 ★). This immediately tells the user if the store is generally a high or low performer.

Total Reviews: The total review count for context (e.g., 154 Reviews).

Top Negative Theme: The most common negative theme for that specific location (e.g., Top Issue: Staff Attitude). This is the most powerful piece of data, confirming if the current complaint is a known, recurring problem.

Scenario B: When a PRODUCT Review is Selected (Unchanged)
The product snapshot remains the same as it is already focused on product-specific data.

Title: Product Snapshot: [Product Name] (e.g., "Product Snapshot: Winter Coat")

Content Layout: A two-column layout.

Data Points:

Product Image: A small thumbnail of the product.

SKU / ID: The product's unique identifier (e.g., SKU: WC-202).

Overall Rating: The product's average star rating (e.g., 4.1 ★).

Top Negative Theme: The most common complaint about that specific product (e.g., Top Issue: Sizing issues).

Updated User Workflow
A user in the Inbox clicks on a new 1-star review.

The reply panel slides in from the right.

The user first sees the customer's name and the 1-star rating.

The new "Location Snapshot" card is immediately visible. They instantly see the location is "Ankara Kızılay," its overall rating is a poor 3.1 ★, and its top negative theme is "Staff Attitude".

They then read the review text below the snippet, which is a complaint about a rude staff member.

Result: The workflow is now even faster and more focused. The user gets all the necessary context to confirm this is a recurring issue at the location without any potentially irrelevant information. They can proceed to write an effective reply with full confidence.
