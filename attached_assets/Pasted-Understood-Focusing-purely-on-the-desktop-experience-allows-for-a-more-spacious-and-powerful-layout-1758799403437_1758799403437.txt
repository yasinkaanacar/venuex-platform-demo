Understood. Focusing purely on the desktop experience allows for a more spacious and powerful layout.

Here is the detailed design and functionality specification for the desktop-only filter bar on the Inbox page.

Filter Bar: Design & Functionality Report (Desktop)
1. Core Principle: Instant & Effortless
The design is optimized for power users on a desktop. There is no "Apply" button. Every filter selection instantly updates the review list below (<500ms), providing immediate feedback and allowing for rapid data exploration.

2. Positioning and Layout
The filter bar is a single, clean horizontal container positioned directly above the review list within the Inbox tab. Its fixed, single-row layout ensures all controls are visible and accessible at all times without wrapping or collapsing. It is visually separated from the content below by a subtle border.

3. Component Breakdown & Styling
The bar consists of four key filter components, arranged logically from left to right for a natural workflow.

Component 1: Location Filter
Purpose: This is the most critical filter, allowing a Regional Manager to drill down into a specific store's performance or an Admin to view the entire organization.

Component Type: A standard dropdown menu (<select>).

Styling:

Clean, modern design with a light gray border (#e2e8f0) and rounded corners.

A subtle chevron (down-arrow) icon on the right.

Default Option: "All Locations", which is the default view for an Admin or Regional Manager. For a Store Manager, this will default to their specific location and be locked.

Component 2: Reply Status
Purpose: The primary workflow filter. The most common user action is to find and respond to unanswered reviews.

Component Type: A pill-style button group. This is faster and more visually clear than a dropdown for a small set of critical options.

Styling:

A set of three connected buttons.

Default State: Light gray background (#f1f5f9), dark gray text.

Active State: The selected button will have a high-contrast style: dark slate background (#1e293b) with white text. This makes the current view immediately obvious.

Buttons & Default Selection:

Unreplied - This is the default selection when the page loads.

Replied

All

Component 3: Filter by Rating
Purpose: Allows users to quickly isolate their most critical (1-2 star) or most positive (4-5 star) feedback.

Component Type: A dropdown menu.

Styling: Consistent with the Location Filter dropdown.

Options:

"All Ratings"

"1-2 Stars ★"

"3 Stars ★"

"4-5 Stars ★"

Component 4: Filter by Theme
Purpose: A powerful feature that allows users to investigate specific recurring issues identified by the lean analytics.

Component Type: A dropdown menu.

Styling: Consistent with the other dropdowns.

Options:

"All Themes"

The list will be dynamically populated with the top themes identified in the Overview module, such as:

"Flavor"

"Staff Attitude"

"Price"

"Cleanliness"

Example User Workflow
A Regional Manager wants to investigate why "Staff Attitude" was a top negative theme this week at their most problematic store, Ankara Kızılay.

They select "Ankara Kızılay" from the Location Filter dropdown.

The Unreplied filter is already active by default.

They select "1-2 Stars ★" from the Filter by Rating dropdown.

They select "Staff Attitude" from the Filter by Theme dropdown.

Result: The review list instantly updates to show only the 1 and 2-star unanswered reviews about staff attitude at the Ankara Kızılay location. The manager has drilled down from thousands of reviews to the 2-3 that require their immediate attention, all in four clicks.
