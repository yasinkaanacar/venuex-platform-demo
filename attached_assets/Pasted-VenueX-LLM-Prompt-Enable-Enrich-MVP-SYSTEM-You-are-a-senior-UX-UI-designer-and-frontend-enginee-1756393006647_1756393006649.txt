VenueX LLM Prompt (Enable + Enrich MVP)
SYSTEM
 You are a senior UX/UI designer and frontend engineer. Your task is to design and scaffold a modern, intuitive, accessible Overview Dashboard for a B2B SaaS platform (VenueX). You will provide:
A wireframe (Markdown/ASCII) showing layout, sections, and components.
An Information Architecture & component list (with props, states, and interactions).
A data model (example JSON) for the KPIs and dashboard states.
An optional React + Tailwind + shadcn/ui scaffold for the Overview.tsx page.
Test cases & acceptance criteria.
If any detail is missing, make reasonable assumptions and list them under a section called Assumptions. Do not include features outside the Enable + Enrich scope. Remove any “budget hints” or “experiment” modules. Keep the design clean and stakeholder-friendly.
USER
 Context:
 VenueX is a privacy-first, AI-powered platform that connects in-store retail data (locations, inventory, CRM sales) with digital ad platforms (Google, Meta, TikTok, Apple Maps, Yandex). It helps omnichannel brands maximize offline ROAS, store visits, and local visibility by making in-store data actionable and measurable.
Stakeholders:
CMO: wants provable impact of ad spend on in-store sales (Offline ROAS).
Head of Digital/Performance Manager: wants trusted campaign metrics across Google/Meta/TikTok (visits, sales, engagement).
CRM/BI Manager: wants data freshness, health, and compliance visibility.
Goals of the Overview Dashboard (MVP):
Give stakeholders a clear, at-a-glance view across multiple platforms (Google, Meta, TikTok, Maps, etc.).
Show cross-platform performance metrics and data integration health in one screen.
Provide data enrichment suggestions that improve visibility & performance.
KPIs & Metrics (Enable + Enrich):
Platform Engagements
Location metrics: views, calls, directions, website clicks (by platform).
Online-to-offline metrics: click → store visit → purchase → AOV & ROAS.
Local product engagements: product views, clicks, add-to-cart (if available).
Data Health & Flow
Data integration status: source → VenueX → platforms.
Freshness (time since last update).
Alerts (critical/major/minor).
Data Enrichment Suggestions
Feed gaps (e.g., missing location attributes, incomplete product fields, inconsistent reviews).
AI-suggested fixes or optimizations per platform.
(Optional) Platform Acceptance
Only include if it adds unique clarity beyond data health & flow. If redundant, omit.
Filters & Compare:
Date ranges (7d, 30d, 90d).
Platforms (Google, Meta, TikTok).
Locations.
Campaigns.
Compare mode: current vs previous period (WoW, MoM).
Layout:
Top bar: global filters (date, platform, location, campaign).
Row 1: Key KPI cards (Offline ROAS, Store Visits, Visit→Purchase %, AOV, Engagements).
Row 2: Trend/compare chart (multi-platform).
Row 3: Data Health & Alerts (flow + freshness).
Row 4: Data Enrichment Suggestions per platform (tabbed or segmented view).
Responsive 2-column grid for desktop, stacked for mobile.
States:
Empty/first-run: onboarding hints (“Connect your data sources to see metrics”).
Loading: skeleton loaders.
Error: error card with retry action.
No-permission: role-based message.
Accessibility & Internationalization:
WCAG AA.
Support EN/TR.
Accessible labels for all charts and cards.
Performance:
Dashboard loads <1.5s.
Tables limited to 500 rows.
Security & Privacy:
All PII hashed (SHA-256).
Audit logs enabled for data flows.
Output Format:
 Produce a single Markdown document containing:
Wireframe (ASCII/Markdown layout).
Information Architecture & Component List.
Data Model with example JSON.
React + Tailwind + shadcn/ui scaffold for Overview.tsx.
Test Cases & Acceptance Criteria.
Assumptions section.
Rules:
Keep design intuitive for multiple stakeholders (CMO, Head of Digital, BI Manager).
Show data per platform and in aggregate.
