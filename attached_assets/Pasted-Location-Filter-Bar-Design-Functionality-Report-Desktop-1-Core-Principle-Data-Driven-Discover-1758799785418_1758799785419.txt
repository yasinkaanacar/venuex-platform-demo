Location Filter Bar: Design & Functionality Report (Desktop)
1. Core Principle: Data-Driven Discovery
The purpose of this filter bar is to transform a simple list of locations into an analytical tool. It allows users to move beyond just viewing data to actively querying their location hierarchy to find performance hotspots and areas needing attention. Like all other filters, it will be instant, updating the location list immediately with each selection.

2. Positioning and Layout
The filter bar is a single, horizontal container positioned at the top of the Locations page, directly under the page title. It provides a persistent set of controls for the data table or map view presented below. The layout is clean and logically ordered to guide the user from broad geographic selections to specific performance metrics.

3. Component Breakdown & Styling
The bar consists of three primary groups of filters, designed to work together to answer complex questions.

Component Group 1: Geographic Hierarchy
Purpose: Allows users to drill down from a national view to a specific city or region. This is the primary method for segmenting the location list.

Component Type: A set of two dependent dropdown menus.

Styling: Consistent with the clean, modern dropdown style used in the Inbox, with light gray borders and rounded corners.

Components:

Region Filter:

Label: Region

Default Option: "All Regions"

Functionality: When a region is selected (e.g., "Marmara Region"), the City Filter options are dynamically updated to show only cities within that region.

City Filter:

Label: City

Default Option: "All Cities"

Component Group 2: Performance Metrics
Purpose: Enables users to find and isolate locations based on key performance indicators (KPIs). This is how a manager finds their best and worst-performing stores.

Component Type: Two distinct dropdown menus for the most important metrics.

Styling: Consistent dropdown style.

Components:

Filter by Average Rating:

Label: Avg. Rating

Options: Provides intuitive performance buckets.

"All Ratings"

"Excellent (4.5+ ★)"

"Good (4.0 - 4.4 ★)"

"Average (3.5 - 3.9 ★)"

"Poor (< 3.5 ★)"

Filter by Reply Rate:

Label: Reply Rate

Options: Focuses on operational efficiency.

"All Reply Rates"

"Excellent (> 95%)"

"Good (80-94%)"

"Needs Improvement (< 80%)"

Component Group 3: Direct Search
Purpose: Provides a "fast path" for users who already know which location they are looking for.

Component Type: A text input field.

Styling:

Positioned to the far right of the filter bar.

A magnifying glass icon inside the input field for clarity.

Placeholder Text: "Search by location name..."

Functionality: The location list updates in real-time as the user types, showing only locations whose names match the search query.

Example User Workflow
An Admin wants to identify all the poorly-rated stores in the Aegean Region that are also failing on their operational goal of an 80% reply rate.

They select "Aegean Region" from the Region Filter dropdown. The City Filter updates accordingly, but they leave it as "All Cities" to see the whole region.

They select "Poor (< 3.5 ★)" from the Filter by Average Rating dropdown. The list of locations immediately shrinks to show only the handful of underperforming stores in that region.

They then select "Needs Improvement (< 80%)" from the Filter by Reply Rate dropdown. The list refines further.

Result: In three clicks, the Admin has a concise, actionable list of the 1-2 specific locations in the Aegean Region that are struggling with both customer satisfaction and operational responsiveness, allowing them to target their intervention precisely.
