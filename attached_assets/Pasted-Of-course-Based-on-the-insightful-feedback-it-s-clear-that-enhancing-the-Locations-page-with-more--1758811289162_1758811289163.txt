Here is a formal design report detailing the new, advanced filter bar for the Locations page, with all content in English.

Design Report: Advanced Location Filtering and Analysis
Version: 1.0

Date: September 25, 2025

Objective: To redesign the filtering capabilities on the Locations page to enable statistically significant and time-based analysis, empowering users to make smarter, data-driven decisions.

1. Design Philosophy & Guiding Principles
This design is built on three core principles:

Data Clarity: To eliminate ambiguity by giving the user explicit control over how metrics (like average rating) are calculated and for what period.

Actionable Insights: To provide tools that allow users to move beyond simple data viewing and ask complex business questions, leading to actionable outcomes.

User Control: To offer the flexibility to switch between different analytical models (e.g., recent performance vs. long-term trends) to suit different business needs.

2. Overall Page Layout & Filter Bar Positioning
The filter bar will be positioned at the top of the Locations page, directly beneath the page title. This ensures the main controls for the data table are always visible. The design is focused on a desktop experience, presenting all filter components in a single, logical horizontal row.

3. Filter Bar: Component Breakdown & Functionality
The redesigned filter bar introduces new, powerful controls to address the core user feedback.

Component 1: Calculation Method & Date Range
Purpose: To define the "how" and "when" of the analysis. These are the most critical controls and are placed first.

Components:

Calculation Method (New!):

Location: Far left of the filter bar.

Component Type: A two-option pill-style toggle group. This design provides a clear, immediate choice between the two fundamental analysis modes.

Style: The active button will be high-contrast (dark background, white text) to make the current mode obvious.

Options:

Periodic Performance (Default): In this mode, all metrics are calculated based only on reviews received within the selected date range. (Answers: "How did we perform in July?").

Overall Trend: In this mode, metrics show a snapshot of the location's all-time data as of the end date of the selected range. (Answers: "Where did our overall reputation stand at the end of July?").

Date Range Filter:

Location: Immediately to the right of the Calculation Method.

Component Type: Standard Date Range Picker.

Functionality: Offers presets like "Last 7 Days," "Last 30 Days," and a custom range selector. Its meaning is dependent on the Calculation Method selected. An interactive tooltip can clarify this for the user.

Component 2: Data Quality Filter
Purpose: To ensure statistical significance by filtering out locations with too little data to be meaningful.

Component:

Minimum Reviews (New!):

Location: To the right of the date filter.

Component Type: A labeled number input field.

Style: A clean, modern text box consistent with other UI elements.

Label: Min. Reviews

Default Value: 20. This provides a sensible starting point for most analyses, and the user can easily change it.

Component 3: Standard Location Filters
Purpose: To allow users to further narrow the list by geography or performance tier.

Components:

Region/City Filters: Standard dropdown menus for Region and City.

Average Rating Filter: A dropdown menu to segment by Avg. Rating using clear buckets (e.g., "< 3.5 ★", "4.0 - 4.4 ★").

4. Interaction and Logic
Instant Update: Any change to a filter will instantly update the location list below. There is no "Apply" button.

Dependent Logic: The Calculation Method toggle is the master control. It fundamentally changes how the data in the table is calculated. The UI should reflect this. For example, the "Average Rating" column header could dynamically change to "Periodic Avg. Rating" or "Overall Avg. Rating" based on the selection.

5. Example User Scenario
Scenario: A Regional Manager wants to identify which of their locations showed the most improved performance within the last quarter, focusing only on locations with significant review volume.

Navigate: They go to the Locations page.

Set Timeframe: They select "Last 90 Days" from the Date Range filter.

Choose Analysis Method: They confirm the Calculation Method is set to the default, Periodic Performance, as they are interested in recent activity, not the all-time average.

Ensure Data Quality: They change the Min. Reviews filter to 50 to focus on their higher-traffic stores.

Analyze: They sort the resulting list by the "Periodic Avg. Rating" column in descending order.

Result: The manager now has a clean, statistically relevant, and actionable list. It shows them which locations performed best specifically within the last quarter, allowing them to identify and reward recent positive turnarounds, independent of the stores' all-time historical data.
