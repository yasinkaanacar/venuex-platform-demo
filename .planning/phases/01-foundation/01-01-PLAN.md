    - Does each navigation produce a POST to `eu.i.posthog.com` containing `$pageview`?
    - Is it firing exactly once per navigation (not zero, not double)?

    **Step 2a — If auto-capture works (most likely):**
    No code changes needed to main.tsx or App.tsx. The current config (no `capture_pageview` key = defaults to `true`) is correct. Document the finding.

    **Step 2b — If auto-capture does NOT fire on route changes:**
    Apply the fix per D-05 and D-06:

    1. In `client/src/main.tsx`, add `capture_pageview: false` to the posthog.init options:
    ```typescript
    posthog.init("phc_4zYXhobdCe3U5MtVSkgGgBFYbl8gBCO4TtCt20YgoBC", {
      api_host: "https://eu.i.posthog.com",
      capture_pageview: false,
      defaults: "2025-01-01",
    });
    ```

    ```typescript
      posthogInstance?.capture('$pageview');
    }, [location, posthogInstance]);
    ```
